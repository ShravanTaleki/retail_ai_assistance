# Retail AI Assistance

A professional, advanced AI-driven personalized retail assistant. This application demonstrates a highly reliable **Hybrid Intelligence** approach, combining strict deterministic data filters with the creative reasoning capabilities of open-weights Local LLMs.

## 🚀 High-Level Overview

This project transforms a standard AI recommendation script into a high-fidelity, fully functional e-commerce storefront. By bridging Python's data-crunching capabilities (Pandas) with a local Ollama reasoning model (`qwen2.5:1.5b`), the application delivers deeply personalized, context-aware product recommendations without suffering from the hallucinations typical of pure LLM workflows.

## 🧠 Architecture Flow

The system operates on a linear, highly controlled data pipeline:

1. **Data Extraction**: Resolves the shopper's profile, including their budget, primary interests, favorite colors, and location.
2. **Pandas Filtering**: Deterministic rule-based tools (found in `tools/`) retrieve candidate products, ensuring pricing, category constraints, and local trend relationships are strictly mathematically accurate.
3. **LLM Reasoning**: The candidate pools are fed into the LLM via prompt engineering. The LLM's *only* task is to select the top candidates and write creative, hyper-personalized reasoning comparing the product strictly to the user's Shopper DNA.
4. **Streamlit UI**: The raw Markdown generated by the LLM is intercepted and parsed. It's then rendered into a responsive, high-end CSS Grid within the Streamlit frontend.

**Flow:** *(Data Extraction) -> (Pandas Filtering) -> (LLM Reasoning) -> (Streamlit UI)*

## ✨ Key Advanced Features

- **Hybrid Architecture (Rules + LLM)**: Uses Pandas for strict, deterministic data filtering (which entirely prevents product hallucinations or impossible pricing) and a local Ollama 1.5b model for personalized reasoning and narrative synthesis.
- **Advanced Fallback Logic**: The Pandas tools (such as `get_product_recommendations.py`) feature cascading fallback mechanisms to gracefully handle "Empty Result Sets" if exact brand or color matches aren't found in the mock catalog.
- **Few-Shot Prompt Engineering**: The LLM system prompt utilizes a concrete "Camping Gear" example to force strict Markdown formatting, fundamentally preventing the small model from hallucinating data structures or blindly regurgitating core instructions.
- **Ethics & Safety Guardrails**: The orchestrator's system prompt strictly prohibits manipulative sales tactics ("Buy now before it's gone") and discriminatory profiling assumptions, enforcing an objective and helpful tone.
- **High-Fidelity UI**: `app.py` has been completely overhauled with custom, theme-responsive CSS to create a premium "Product Card Grid" storefront. It dynamically adapts to both Streamlit Light and Dark modes with tailored borders, shadows, and crisp typography.
- **Offline & Private**: This ecosystem entirely runs locally. It guarantees absolute privacy without ever sending your proprietary retail catalogs or user behavioral data out to external APIs.

## 🛠️ Setup and Installation

### Prerequisites
- Python 3.10+
- `uv` package manager (recommended)
- [Ollama](https://ollama.com/) (installed and running locally)

### Installation Steps

1. **Clone & Environment Setup**
   Ensure you are in the project root directory, then activate the virtual environment and install dependencies:
   ```bash
   uv venv
   # Windows:
   .venv\Scripts\activate
   # Mac/Linux:
   source .venv/bin/activate
   
   uv pip install -r requirements.txt
   ```

2. **Generate Synthetic Demo Data**
   Run the data generation script to build the local CSV catalogs:
   ```bash
   uv run python generate_data.py
   ```

3. **Start the Local LLM via Ollama**
   In a separate terminal, pull and start the reasoning model (make sure the Ollama app is running):
   ```bash
   ollama pull qwen2.5:1.5b
   ```

4. **Launch the Storefront**
   Fire up the High-Fidelity UI:
   ```bash
   uv run streamlit run app.py
   ```
   *Navigate to the Local URL (e.g., http://localhost:8501) provided in the terminal to view the storefront.*
